Dicta превращает речь в текст и автоматически вставляет результат в Word, браузер, мессенджер, почту или редактор кода. Вы сами выбираете, сохранить речь дословно или привести её к аккуратному деловому стилю.

Приложение работает в системном трее, вызывается глобальной горячей клавишей и использует ваши API-ключи Groq, OpenAI, OpenRouter или Polza.ai.

## Основные возможности

- **Голосовой ввод в любом приложении** — глобальная горячая клавиша запускает и останавливает запись, даже когда окно Dicta скрыто.
- **Автовставка результата** — готовый текст копируется в буфер обмена и вставляется в активное поле.
- **Раздельный выбор STT и LLM** — провайдер и модель распознавания могут отличаться от провайдера и модели постобработки.
- **Автоматическое определение языка** — STT-модель может определить язык речи без принудительной русской подсказки.
- **Проверка API-ключей** — Dicta проверяет подключение провайдера и сохраняет статус успешной проверки.
- **Выбор и проверка микрофона** — можно выбрать устройство, прослушать его и увидеть живой уровень сигнала.
- **Настраиваемая горячая клавиша** — поддерживаются одиночные клавиши и сочетания с `Ctrl`, `Alt`, `Shift` и `Win`.
- **Плавающий виджет** — показывает запись, обработку, ошибку и успешное копирование, не забирая фокус у рабочего приложения.
- **Локальная история** — сохраняет исходный и обработанный текст, позволяет редактировать записи и сравнивать изменения.
- **Повтор после сетевой ошибки** — аудио временно остаётся в памяти, поэтому не нужно диктовать текст заново.
- **Командная палитра** — быстрый поиск по действиям и истории, переключение режима и темы через `Ctrl+K`.
- **Автозапуск и системный трей** — Dicta может запускаться вместе с Windows и оставаться доступной в фоне.

## Режимы обработки текста

Dicta предлагает четыре уровня обработки. Режим можно менять в настройках или через командную палитру.

| Режим | Что делает | Когда использовать |
|---|---|---|
| **Без обработки** | Возвращает дословную транскрипцию без LLM | Цитаты, заметки, расшифровка речи |
| **Лёгкая правка** | Исправляет пунктуацию, опечатки и явные повторы | Повседневные сообщения |
| **Сбалансированный** | Убирает слова-паразиты и заминки, делает текст связным, сохраняя смысл и стиль | Универсальный режим |
| **Деловой стиль** | Преобразует речь в вежливый и профессиональный текст без сокращения содержания | Письма, отчёты, рабочая переписка |

## Персонализация

- **Системная, светлая и тёмная темы** — системный режим следует оформлению Windows.
- **Собственный акцентный цвет** — цвет кнопок, выделений и активных элементов выбирается через палитру.
- **Прозрачность акцента** — насыщенность цветовых заливок регулируется отдельно.
- **Сохранение оформления** — тема, цвет и прозрачность синхронизируются между главным окном и виджетом.

## Провайдеры и модели

| Провайдер | Распознавание речи | ИИ-постобработка |
|---|---:|---:|
| **Groq** | Да | Да |
| **OpenAI** | Да | Да |
| **OpenRouter** | Да | Да |
| **Polza.ai** | Да | Да |

В расширенных настройках можно отдельно выбрать STT-провайдера, STT-модель, LLM-провайдера и LLM-модель. Dicta предупреждает, если для выбранного сервиса отсутствует ключ или сохранённая модель больше недоступна.

## Скачать и установить

Откройте [последний релиз](https://github.com/gajsin/dicta/releases/latest) и скачайте один из файлов:

- `Dicta_0.2.0_x64-setup.exe` — обычный установщик Windows, рекомендуется большинству пользователей;
- `Dicta_0.2.0_x64_en-US.msi` — MSI-пакет для ручного или корпоративного развёртывания.

После установки:

1. Запустите Dicta и откройте раздел «Провайдеры».
2. Добавьте собственный API-ключ и нажмите «Сохранить и проверить».
3. Выберите микрофон, режим обработки, STT- и LLM-модели.
4. При желании настройте горячую клавишу, тему и акцентный цвет.
5. Нажмите `F9`, произнесите текст и снова нажмите `F9`.

Сборки пока не подписаны цифровым сертификатом. Windows SmartScreen может показать предупреждение при первом запуске.

## Конфиденциальность и данные

- API-ключи и настройки хранятся только локально на устройстве пользователя.
- История транскрипций хранится локально в WebView-хранилище и может быть отключена.
- Аудио отправляется только выбранному STT-провайдеру.
- Текст отправляется LLM-провайдеру только при включённой постобработке.
- После успешной обработки аудио удаляется из памяти; при ошибке оно временно сохраняется для повтора.
- Dicta не имеет собственного сервера и не получает пользовательские ключи, аудио или историю.

Подробнее: [SECURITY.md](SECURITY.md).

## Как устроен процесс

```text
Горячая клавиша
      ↓
Запись с выбранного микрофона
      ↓
Ресемплинг в mono 16 kHz WAV
      ↓
Распознавание речи (STT)
      ↓
Выбранный режим обработки (опционально LLM)
      ↓
История → буфер обмена → вставка в активное приложение
```

## Сборка из исходников

### Требования

- Windows 10 или Windows 11;
- [Node.js](https://nodejs.org/) 22.12 или новее;
- [Rust](https://www.rust-lang.org/tools/install), стабильная версия;
- Visual Studio C++ Build Tools;
- Microsoft Edge WebView2 Runtime.

### Разработка

```powershell
npm ci
npm run tauri dev
```

### Проверка

```powershell
npm run check
npm run format:check
npm run lint
```

### Релизная сборка

```powershell
npm run tauri build
```

Установщики появятся в `src-tauri/target/release/bundle/`.

## Архитектура

| Слой | Назначение |
|---|---|
| Tauri 2 / Rust | Окна, трей, горячие клавиши, запись аудио, настройки и сетевые запросы |
| Svelte 5 / TypeScript | Интерфейс, состояние записи, история и настройки |
| `cpal` + `rubato` | Захват, мониторинг и ресемплинг аудио |
| `reqwest` | Запросы к STT- и LLM-провайдерам |

```text
src/
├── App.svelte                # Композиция окон и представлений
└── lib/
    ├── components/           # Интерфейс
    ├── recording/            # Состояния записи и обработки
    ├── settings/             # Модель и синхронизация настроек
    └── history/              # Локальная история

src-tauri/
├── src/
│   ├── audio/                # Захват, мониторинг, WAV и ресемплинг
│   ├── commands/             # IPC-команды Tauri
│   ├── transcription/        # STT и LLM
│   ├── config.rs             # Валидация и хранение настроек
│   └── shortcut.rs           # Глобальная горячая клавиша
└── tauri.conf.json
```

Подробное описание: [docs/architecture.md](docs/architecture.md).
