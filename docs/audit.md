# Результаты технического аудита

Аудит выполнен по актуальному локальному проекту. Удалённый GitHub-репозиторий не использовался как источник кода.

## Исправленные классы проблем

- реактивный откат API-ключа до завершения сохранения;
- отсутствие обратной связи при ошибке backend-сохранения;
- расхождение отображаемой и реально зарегистрированной hotkey;
- ограниченный набор клавиш и зависимость букв от раскладки;
- небезопасное восстановление повреждённого файла настроек;
- несогласованные frontend/backend defaults;
- случайное попадание пользовательского `settings.json` в репозиторий или отчёты;
- конкурентные и устаревшие снимки настроек;
- удержание успешно обработанного аудио в памяти;
- некорректный выбор записи при фильтрации истории;
- вымышленные значения длительности вместо нуля;
- утечки таймера предупреждения микрофона;
- отсутствие выходного аудиопотока и скрытая ошибка запуска при проверке микрофона;
- пропуск состояния «Скопировано» на главном экране и отдельная несинхронизированная кнопка при пустой истории;
- обрезание статуса провайдера на рабочей ширине окна настроек;
- перегруженные компоненты и неиспользуемый UI-компонент;
- тихий «успех» сохранения при недоступном пути `%APPDATA%` и отсутствие отката runtime-настроек при ошибке записи;
- гонка быстрого запуска/остановки проверки микрофона;
- одноразовая гонка загрузки backend-настроек при раннем старте главного WebView;
- изменение ширины рабочей области и одновременное участие двух record-состояний в layout во время `fade`;
- лишнее подтверждение при удалении одной записи;
- известные уязвимости старой dev-цепочки Vite/esbuild; зависимости обновлены, итоговый `npm audit` — 0 уязвимостей.
- преждевременный показ главного окна до применения сохранённой геометрии и готовности frontend;
- повторное центрирование окна при открытии из трея, конфликтовавшее с сохранённой позицией;
- отсутствие результата проверки провайдеров в `settings.json`;
- отдельное хранение акцентного цвета в Local Storage вместо общей модели настроек;
- русский language prompt при автоматическом распознавании речи у OpenAI-совместимых провайдеров;
- выбор OpenRouter-модели, совмещающей транскрипцию и перевод, как единственной и используемой по умолчанию.
- внутреннее значение прозрачности color picker, которое не попадало ни в callback, ни в `settings.json`, ни в CSS;
- синхронная сериализация и замена `settings.json` на каждом `Moved`/`Resized`, блокировавшая drag главного окна и overlay;
- разные `fit-content`-размеры капсулы для записи, обработки и успешного копирования;
- покадровое изменение `height` полос waveform, вызывавшее layout во время записи и перетаскивания виджета.

## Границы проверки

Автоматические тесты не выполняют реальные запросы к сторонним API. Они проверяют локальную запись и повторное чтение ключей из `settings.json`; доступность провайдера проверяется приложением при нажатии «Сохранить и проверить» и требует пользовательского ключа.
