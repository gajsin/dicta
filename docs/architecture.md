# Архитектура Dicta

## Слои

- `src/App.svelte` — композиция окон и маршрутизация представлений.
- `src/lib/settings` — нормализация настроек, очередь сохранения и контроллер синхронизации с Tauri.
- `src/lib/recording` — state machine записи, распознавания, вставки и overlay.
- `src/lib/history` — локальная история и проверка сохранённых данных.
- `src/lib/components` — представление без backend-логики.
- `src-tauri/src/commands` — узкий IPC-слой.
- `src-tauri/src/audio` — устройства, захват, живой мониторинг микрофона, ресемплинг и WAV.
- `src-tauri/src/transcription` — STT и LLM-провайдеры.
- `src-tauri/src/config.rs` — валидация и атомарная запись локальных настроек.
- `src-tauri/src/settings_persistence.rs` — единый фоновый writer с debounce и принудительным flush.
- `src-tauri/src/shortcut.rs` — канонизация и атомарная замена глобальной горячей клавиши.

## Настройки и секреты

Frontend отправляет единый снимок настроек через `sync_settings`. Backend валидирует значения, применяет hotkey и автозапуск, затем атомарно сохраняет параметры и API-ключи в `%APPDATA%\Dicta\settings.json`.

Запись `settings.json` выполняется через временный файл. Частые события перемещения и изменения размера обновляют состояние в памяти, а отдельный writer сохраняет только последнюю геометрию после 250 мс покоя. Закрытие, скрытие и обычное изменение настроек вызывают упорядоченный flush через тот же writer, поэтому drag не блокируется файловым I/O и параллельные записи не перетирают друг друга. Повреждённый JSON не замалчивается: при запуске создаётся резервная копия `settings.invalid-<timestamp>.json`, после чего восстанавливаются безопасные значения по умолчанию.

В desktop runtime единственным источником настроек является локальный backend-файл. Local Storage используется только в браузерном preview без Tauri и никогда не читается или записывается EXE. Главный WebView повторяет первоначальную загрузку при кратковременной недоступности Tauri state.

Результаты успешной проверки провайдеров хранятся в `verifiedProviders`. При замене или удалении ключа соответствующий статус автоматически сбрасывается. Акцентный цвет и его прозрачность (`accentColor`, `accentOpacity`) также являются частью общей модели `settings.json`; отсутствие нового поля в старом файле обратно совместимо и означает 100%.

## Запуск главного окна

Главное окно создаётся скрытым. Rust сначала загружает `settings.json` и применяет сохранённые координаты и размер. Svelte гидратирует backend-настройки и только после готового кадра вызывает `reveal_main_window`. Поэтому пользователь не видит конфигурационный размер, пустой WebView или последующий скачок геометрии.

Повторное открытие из трея сохраняет текущую позицию окна и не вызывает принудительное центрирование.

Overlay всегда имеет постоянный нативный размер 440×80, а видимая капсула — 264×42 во всех состояниях. Смена «Слушаю» → обработка → «Скопировано» затрагивает только содержимое через `opacity`/`transform`, поэтому центр и внешняя геометрия не прыгают.

## Автоматический язык STT

При `selectedLanguage: "auto"` поле `language` не отправляется, поэтому язык определяет сама STT-модель. Language-specific prompts не добавляются ни одному провайдеру. Для OpenRouter моделью по умолчанию служит специализированная `openai/gpt-4o-mini-transcribe`; Whisper Large V3 остаётся доступен как ручной вариант.

## Горячая клавиша

UI использует `KeyboardEvent.code`, поэтому раскладка клавиатуры не меняет физическую клавишу. Backend канонизирует строку и перед заменой проверяет регистрацию нового shortcut. Поддерживаются буквы, цифры, F1–F24, знаки, навигационные, lock-, numpad- и media-клавиши, а также сочетания с `Ctrl`, `Alt`, `Shift` и `Win`.

## Проверка микрофона

Тест микрофона запускает связанные входной и выходной потоки `cpal` на общей частоте дискретизации. Небольшой предварительный буфер сглаживает джиттер, а ограниченная очередь не позволяет задержке расти при перегрузке. Параллельно frontend получает RMS-уровень для шкалы громкости.

## Проверки

Публичный CI на Windows проверяет TypeScript/Svelte, production build, `rustfmt` и Clippy. Расширенный внутренний набор проверок хранится отдельно от публичного репозитория.
